# Milestones

## M1 基础骨架

状态：已完成

目标：

- 初始化 `go.mod`
- 建立基础目录结构
- 提供最小 CLI 或 demo 入口
- 定义面向 Lua 5.1 子集的 `State`、`Value`、`Stack` 基础对象

交付标准：

- 项目可以编译
- 可以创建 VM 实例
- 可以跑一个最小演示入口

## M2 加载链路

状态：已完成

目标：

- 实现面向 Lua 5.1 子集的 lexer
- 实现面向 Lua 5.1 子集的 parser
- 生成 AST 或 IR
- 打通从源码到内部表示的转换

交付标准：

- 能读取并解析简单 Lua 脚本
- 能输出结构化的内部表示

## M3 执行核心

状态：已完成

目标：

- 实现 Lua 5.1 子集基础指令或节点执行
- 完成栈、帧、PC 管理
- 支持基础表达式与流程控制

交付标准：

- 能执行简单算术、变量、条件、循环脚本

## M4 运行时扩展

状态：进行中

目标：

- 支持 Lua 5.1 子集所需的函数调用
- 支持基础全局环境
- 提供最小宿主函数注册机制
- 逐步补充 table 或闭包能力

交付标准：

- Go 可以注入函数
- Lua 可以完成基本函数调用

当前拆分阶段：

- `M4.5 语义收口`：进行中
- `M4.6 标准库扩展`：待开始

### M4.5 语义收口

状态：进行中

目标：

- 系统收口 Lua 5.1 多返回值边界语义
- 继续推进 `#table` / `table.getn` / 长度相关行为
- 收口 metatable 的关键比较、索引与回退规则
- 扩 generic `for` 的迭代器兼容性和边界验证

交付标准：

- 多返回值不再只依赖零散 case 支撑
- `#table` 和长度相关行为比当前更接近 Lua 5.1
- metatable 关键比较和回退链路再收一轮
- generic `for` 不再只在主链路上“能跑”

### M4.6 标准库扩展

状态：待开始

目标：

- 扩 `string` 库，优先 pattern / capture 相关能力
- 扩 `table` 库，补剩余高频辅助函数和边界
- 扩 `math` 库，补剩余常用函数和行为细节
- 校对 builtin 的参数规则、返回值和错误路径

交付标准：

- `string` 不再只是纯文本最小子集
- `table` / `math` 从“小子集”提升到“常用子集较完整”
- 常见 Lua 5.1 脚本不再频繁卡在标准库空白上
